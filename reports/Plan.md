# План автоматизации
План автоматизации тестирования комплексного сервиса по покупке туров, взаимодействующего с СУБД и API Банка.

### Перечень автоматизируемых сценариев
Покупка тура через оплату по карту (через кнопку «Купить»):
Позитивные сценарии:
-	Успешная покупка тура с помощью карты со статусом "APPROVED":
1.	Открыть страницу сервиса по покупке туров;
2.	В поле Номер карты ввести «4444 4444 4444 4441»;
3.	В поле Месяц ввести валидные значения месяца (формат мм) и года (формат гг) не ранее текущей даты. Пример: месяц - значение «05», «Год» – «23»;
4.	В поле Владелец ввести любую валидную фамилию и имя на латинице. Пример - «Ivanov Ivan»;
5.	В поле CVC/CVV ввести любое трехзначное число, за исключением 000;
6.	Кликнуть ЛКМ по кнопке «Продолжить»;
      Ожидаемый результат: на экране в правом верхнем углу отобразилось уведомление «Успешно Операция одобрена Банком.».

-	Ошибка при оплате тура с помощью карты со статусом "DECLINED":
1.	Открыть страницу сервиса по покупке туров;
2.	В поле Номер карты ввести «4444 4444 4444 4442»;
3.	В поле Месяц ввести валидные значения месяца (формат мм) и года (формат гг) не ранее текущей даты. Пример: месяц - значение «11», «Год» – «22»;
4.	В поле Владелец ввести любую валидную фамилию и имя на латинице. Пример - «Ivanov Ivan»;
5.	В поле CVC/CVV ввести любое трехзначное число, за исключением 000;
6.	Кликнуть ЛКМ по кнопке «Продолжить»;
      Ожидаемый результат: на экране в правом верхнем углу отобразилось уведомление «Ошибка! Банк отказал в проведении операции.».

      b.	Поле оставить пустым;

3.	В поле Месяц ввести валидные значения месяца (формат мм) и года (формат гг) не ранее текущей даты. Пример: месяц - значение «11», «Год» – «22»;

      a.	на экране в правом верхнем углу отобразилось уведомление «Ошибка! Банк отказал в проведении операции.».

      c.	поле выделено красным цветом. Под полем отображается сообщение «Неверный формат»
      d.	поле выделено красным цветом. Под полем отображается сообщение «Неверный формат»

Месяц
1.	Открыть страницу сервиса по покупке туров;
2.	В поле Номер карты ввести «4444 4444 4444 4441»;
3.	В поле Месяц ввести:
      a.	Поле оставить пустым;
      b.	Вести одну цифру. Пример – «3»;
      c.	Ввести значение «00»
      d.	Ввести значение больше 12. Пример «13».
4.	В поле Год ввести валидные значения года (формат гг) равным или больше текущего года.
5.	В поле Владелец ввести любую валидную фамилию и имя на латинице. Пример - «Ivanov Ivan»;

7.	Кликнуть ЛКМ по кнопке «Продолжить»;

### Перечень используемых инструментов
IntelliJ IDEA – удобная интегрированная среда разработки программного обеспечения для многих языков программирования. Имеется опыт работы в данной среде.

Java - строго типизированный объектно-ориентированный язык программирования общего назначения,

Gradle — удобный и простой в использовании инструмент автоматизации сборки и управления зависимостями, который сами выкачивает нужные нам библиотеки.

JUnit5 –наиболее часто используемая библиотека для модульного тестирования программного обеспечения на Java. Имеется опыт работы с данной библиотекой.


Selenide – это фреймворк для автоматизированного тестирования веб-приложений на основе Selenium WebDrive. Имеется опыт работы с данной библиотекой.

MySQL – это реляционная система управления базами данных с открытым исходным кодом. В настоящее время эта СУБД одна из наиболее популярных в веб-приложениях.

Docker – удобное в использовании программное обеспечение для автоматизации развёртывания и управления приложениями в средах с поддержкой контейнеризации, контейнеризатор приложений. Имеется опыт работы с данной библиотекой.

Allure – фреймворк, предназначенный для создания визуально наглядной картины выполнения тестов. Свободно дружит и с JUnit5.

JavaFaker - библиотека генерации случайных данных.


### Риски при автоматизации
Слодности при развертывании сервиса на локальном компьютере.
Сложности при использовании некоторых инструментов при автоматизации из-за недостаточного их изучения.
Сложности с поиском локаторов на странице сервиса.
Ограничение времени.
Ошибки в сервисе, которые могут увеличить время при автоматизации.

### Интервальная оценка с учётом рисков (в часах)

Подготовке отчётных документов по итогам автоматизированного тестирования – 8 часов.
Подготовка отчётных документов по итогам автоматизации – 6 часов.
